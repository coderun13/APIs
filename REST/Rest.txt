REST
Representational State Transfer
REST is a architectural style that defines a set of constraints(rules) to be used for creating web services.

RESTful APIs(applies rules of REST)
examples- GET/2/users,GET/2/tweets etc.

Resource- documentation(blogpost on stackoverflowc-- best practices for rest api design)

GET- retrieves resources
POST- submits new data to the server
PUT- updates existing data
PATCH- update existing data partially
DELETE- removes data

Resource - data on which CRUD operation is performed.
